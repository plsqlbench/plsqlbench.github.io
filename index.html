About PLSQLBENCH

PLSQLBENCH is an evaluation framework for procedural database programming. It tests whether models can write PL/SQL artifacts that compile, execute, query database state correctly, and pass task-specific unit tests.

Why PLSQLBENCH?

Production database work is not only standalone SQL. It includes reusable database-resident programs that validate records, coordinate transactions, handle exceptions, and run scheduled jobs.

  • Executable artifacts: procedures, functions, packages, triggers, cursors, and dynamic SQL.
  • Enterprise schemas: broad table sets and realistic database objects.
  • Multi-turn workflows: later turns add requirements, repair failures, or change business rules.

Evaluation Protocol

  1. Generate: models receive a natural-language request and Oracle-dialect schema context.
  2. Compile and run: generated artifacts execute against benchmark schemas.
  3. Score tests: unit tests compare outputs, table-state changes, exceptions, and diagnostics.

Submission Format

Submit one JSONL row per generated answer. Include model metadata, task identifiers, generated PL/SQL, and optional run telemetry. 

{"instance_id":"spider2_singleturn_0001","model":"your-system","generated_plsql":"CREATE OR REPLACE PROCEDURE ...","latency_seconds":8.5}
{"conversation_id":"plsql-multiturn-0732","turn_index":2,"model":"your-system","generated_plsql":"CREATE OR REPLACE PACKAGE ..."}
