Why it matters

A Missing Layer Between Code Generation and Text-to-SQL

Production database work is not only standalone SQL. It includes reusable database-resident programs that validate records, coordinate transactions, handle exceptions, and run scheduled jobs.

Executable database programs

PLSQLBENCH evaluates stored procedures, functions, packages, triggers, cursors, object types, dynamic SQL, and transaction logic.

Not Oracle-only as a problem class

Procedural SQL equivalents exist across major platforms, including BigQuery scripting, Snowflake Scripting, PL/pgSQL, and Transact-SQL.

Multi-turn enterprise workflows

Later turns add requirements, repair failures, or change business rules while preserving earlier behavior and names.

Evaluation

Execution-Based Protocol

1

Generate PL/SQL

Models receive a natural-language request and Oracle-dialect schema context.

2

Compile and run

Generated artifacts are executed against benchmark schemas in Oracle Autonomous Database.

3

Score tests

Unit tests compare outputs, table-state changes, exceptions, and diagnostic messages.
